There Were No 43 Failures Someone turned @steipete's story into a meme. Sold previous company for $100M+, spend three years in existential crisis, get jacked, come back from retirement, vibe-code 43 failed projects, and then project 44, OpenClaw, goes viral. It got over 11,000 likes. It's a funny meme, but it's not what happened. There were no 43 failures. Peter Steinberger's GitHub account, steipete, has 168 public repositories. His bio reads: "Came back from retirement to mess with AI." If you trace what he actually built after coming back, in chronological order, you don't see random trial and error. You see a precisely engineered modular architecture. In July 2025, he built poltergeist, a universal hot reload tool that automatically triggers builds when files change in any language. The same month, he created Tachikoma, a Swift SDK for accessing any AI model through a single interface. Build automation and multi-model AI access. Both became foundational layers of OpenClaw. In November 2025, the pace accelerated. agent-scripts (1,912 stars) was a shared script library for agents. mcporter (1,857 stars) turned MCP servers into callable TypeScript APIs. oracle (1,465 stars) let you query GPT-5 Pro with custom context and files. CodexBar (5,970 stars) put AI coding agent usage stats in your menu bar. sweetlink (85 stars) connected agents to web apps through browser control. On November 24th, the OpenClaw repository was created. December brought an explosion of skills. gogcli (3,616 stars) was a CLI for Gmail, Google Calendar, Drive, and Contacts. It is now OpenClaw's Google Suite skill. wacli (429 stars) was a WhatsApp CLI. It is now OpenClaw's WhatsApp skill. imsg (696 stars) was an Apple Messages CLI. It is now OpenClaw's iMessage skill. summarize (3,221 stars) turned any URL or YouTube video into a summary. camsnap (55 stars) was a camera snapshot CLI, sag (177 stars) was a modern TTS tool, and brabble (94 stars) was a voice trigger that responded to "Hey, Computer." All of them live inside OpenClaw today. The combined GitHub stars of these "failed projects" exceed 20,000. Failed projects don't individually attract hundreds or thousands of stars. The meme frames it as if luck struck on the 44th attempt. In reality, projects 1 through 43 were the components of project 44. He built the engine, the wheels, the steering wheel, and the seats separately, then assembled them. He didn't crash 43 cars and get lucky on the 44th. Look at the timeline again. July: build automation and AI SDK. November: agent framework and MCP integration. December: messaging, camera, voice, and browser skills, all at once. This is textbook bottom-up systems design. Not vibe-coding. Architecture. This pattern is not new in software. When the Linux kernel first appeared, Linus Torvalds didn't suddenly create an operating system out of nowhere. Before that, he studied MINIX, built a terminal emulator, and experimented with task switching. Each was a small project. Each was a step toward the kernel. What Steinberger did follows the same logic. Each CLI tool worked independently, yet all of them were converging toward a single agent platform. We crave drama in success stories. Repeated failure, despair, and a last-minute twist. That narrative is more entertaining. But most real technical breakthroughs are not dramatic. A person with direction stacks modules one by one, and at some point those modules click together into a system. From the outside, that moment looks like it came out of nowhere. OpenClaw currently has 202,541 GitHub stars. Those 200,000 stars did not materialize from the 44th project alone. The seeds were already planted in repositories 1 through 43. The meme about 43 failures is funny, but what actually happened is more interesting. 43 components became one machine. It was not failure. It was engineered effort.
